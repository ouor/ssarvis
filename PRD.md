# PRD

## 1. 문서 개요

- 문서명: SSARVIS Product Requirements Document
- 문서 목적: 기존 `클론 생성/체험 서비스`에서 `SNS + 사람 간 DM + 사용자 대리 AI 자동응답 서비스`로의 제품 전환 방향을 고정하고, 이후 설계/개발/디자인 참고 기준으로 사용한다.
- 문서 상태: Working Baseline
- 대상 독자: 기획, 개발, 디자인, QA
- 문서 원칙: 이 문서는 설명문이 아니라 결정문이다. 정책, 정의, 범위, 비범위를 명시적으로 기록한다.

## 2. 제품 한 줄 정의

SSARVIS는 사용자가 게시물과 DM으로 소통하는 SNS이며, 각 사용자는 자신을 대리하는 클론 AI를 1개 보유하고 필요 시 DM에서 자동응답하게 할 수 있다.

## 3. 배경과 리브랜딩 방향

- 기존 SSARVIS는 설문 기반 클론 생성, 클론 체험, 클론 간 채팅/논쟁 중심 서비스였다.
- 새로운 SSARVIS는 인스타그램과 유사한 SNS 구조를 기본으로 삼는다.
- 제품의 중심은 더 이상 `클론 그 자체`가 아니라 `사용자 간 관계와 소통`이다.
- 클론은 독립 캐릭터가 아니라 `SNS 계정 사용자를 대리하는 AI 페르소나`로 재정의된다.
- DM은 기본적으로 사람과 사람 사이의 대화이며, 사용자의 설정과 상태에 따라 클론이 대신 응답한다.

## 4. 목표

- 게시물, 프로필, 팔로우, DM을 갖춘 SNS 구조를 제공한다.
- 각 사용자에게 자신을 대리하는 클론 AI 1개와 보이스 1개를 연결한다.
- 사용자가 원할 때 클론이 DM 자동응답을 수행하도록 한다.
- 자동응답이 사람과 AI를 혼동시키지 않도록 명확히 표기한다.
- 기존 SSARVIS의 강점인 클론 생성, TTS, 음성 상호작용을 SNS 맥락으로 재배치한다.

## 5. 비목표

- MVP에서 점진적 학습 자동 반영은 구현하지 않는다.
- MVP에서 실시간 음성 통화는 구현하지 않는다.
- MVP에서 AI 응답 생성 후 복잡한 안전 필터링 시스템은 구현하지 않는다.
- MVP에서 AI가 AI 메시지에 다시 자동응답하는 루프형 대화는 허용하지 않는다.
- MVP에서 다중 클론, 다중 보이스 소유 구조는 유지하지 않는다.
- MVP에서 논쟁 기능은 메인 사용 흐름이 아니다.

## 6. 핵심 용어 정의

- 사용자: SNS 계정을 가진 실제 사람.
- 클론: 특정 사용자를 대리하는 AI 페르소나. 독립 캐릭터가 아니다.
- 보이스: 해당 사용자의 클론이 음성으로 응답할 때 사용하는 대표 음성 자산.
- 공개 계정: 검색 가능하며, 공개 게시물을 누구나 조회할 수 있고 누구나 DM을 보낼 수 있는 계정.
- 비공개 계정: 신규 팔로우 요청이 불가능하며, 이미 팔로우 중인 사용자만 게시물 조회와 DM이 가능한 계정.
- 팔로우: 다른 사용자의 게시물 및 상태에 대한 관계 연결.
- 맞팔로우: 두 사용자가 서로 팔로우 중인 상태. 기존 서비스의 친구 개념을 계승하는 관계적 의미를 가진다.
- DM: 사용자 간 1:1 대화 공간.
- AI 응답: 클론이 사용자를 대신하여 생성한 메시지.
- 자동응답: 사용자의 설정에 따라 클론이 DM에서 대신 답장하는 기능.
- 부재중: 마지막 API 요청 시점으로부터 3분 이상 경과한 상태.
- 활동: 게시물 업로드, 게시물 조회, 대화, 기타 사용자 행동을 포함한 모든 API 요청.
- 음성 대화: 기본 텍스트 DM과 별개로, 음성 메시지를 주고받는 대화 모드.
- AI 숨김: 개인 UI 설정에 따라 특정 AI 응답 묶음을 화면에서 보이지 않게 하는 기능.

## 7. 타깃 사용자와 주요 시나리오

- 자기 표현과 관계 중심의 SNS를 사용하고 싶지만, 부재 시에도 어느 정도 응답 연속성을 유지하고 싶은 사용자
- AI가 자신의 말투와 분위기를 대리해주는 경험에 흥미를 느끼는 사용자
- 텍스트와 음성 메시지를 혼합해 가볍게 소통하고 싶은 사용자

주요 시나리오:

- 사용자는 게시물을 올리고 다른 사용자의 프로필과 게시물을 탐색한다.
- 사용자는 다른 사람에게 DM을 보낸다.
- 상대 사용자가 자동응답을 켜 두었고 부재중이면, 상대 클론이 대신 답장한다.
- 사용자는 나중에 돌아와 하나의 대화창에서 사람 메시지와 AI 메시지를 함께 확인한다.
- 사용자는 원할 경우 AI가 개입한 메시지 묶음을 자신의 화면에서 숨긴다.
- 사용자는 텍스트 DM을 음성으로 재생해 들을 수 있다.
- 사용자는 음성 대화 모드에서 클론과 음성 메시지 형태로 상호작용할 수 있다.

## 8. 제품 원칙

- 사람 간 관계와 대화가 제품의 중심이다.
- AI는 사용자를 보조하거나 대리하지만 사용자 자체를 대체하는 정체성으로 취급하지 않는다.
- AI 개입 사실은 사용자에게 숨기지 않는다.
- SNS로서의 일상 사용성을 우선하고, AI 기능은 이를 보강하는 역할이어야 한다.
- MVP는 작고 명확해야 하며, 깊은 학습 기능보다 확실한 자동응답 경험을 우선한다.

## 9. 기능 범위

MVP 포함:

- 인증
- 프로필
- 게시물 업로드 및 조회
- 사용자 검색
- 팔로우 기반 관계
- 공개/비공개 계정 정책
- 1:1 DM
- 클론 자동응답
- AI 응답 표기
- AI 응답 묶음 숨김
- 클론 1개 설정
- 보이스 1개 설정
- 텍스트 DM의 음성 재생
- 음성 메시지형 대화
- 프로필에서 접근하는 논쟁 기능

MVP 제외:

- 점진적 학습 자동 업데이트
- 실시간 통화형 보이스 콜
- AI 안전 필터 고도화
- 그룹 DM
- 다중 클론/다중 보이스 관리

## 10. 도메인 규칙

- 사용자당 클론은 정확히 1개만 가진다.
- 사용자당 보이스는 정확히 1개만 가진다.
- DM 기본 상대는 사람이다.
- 클론은 사용자의 설정에 따라 DM에서만 대리 응답한다.
- 자동응답 설정 값은 `항상`, `내가 접속중이지 않을 때`, `사용 안함` 세 가지다.
- `내가 접속중이지 않을 때`는 마지막 API 요청 후 3분이 지나면 발동한다.
- 활동 여부 판정은 화면 체류가 아니라 API 요청 발생 기준이다.
- AI는 AI가 생성한 메시지에 자동응답하지 않는다.
- AI 응답은 원본 대화 흐름 내에 저장한다.
- AI 응답은 프로필 사인 우측 하단에 `AI` 마크 등 명확한 시각 요소로 표시한다.
- AI 숨김은 개인 설정이며, 상대방 화면에는 영향을 주지 않는다.
- AI 숨김 단위는 `상대의 유발 메시지 + 해당 AI 응답` 묶음이다.
- 점진적 학습은 장기 방향이지만 MVP 범위에서 제외한다.

## 11. 사용자 흐름

핵심 사용자 흐름은 다음과 같다.

1. 사용자가 계정을 생성하고 프로필을 완성한다.
2. 사용자는 자신의 클론과 보이스를 설정한다.
3. 사용자는 공개 계정 또는 비공개 계정으로 운영한다.
4. 사용자는 게시물을 업로드하거나 다른 사용자의 게시물을 조회한다.
5. 사용자는 다른 사람에게 DM을 보낸다.
6. 수신 사용자가 직접 응답 가능하면 사람이 답장한다.
7. 수신 사용자가 자동응답 상태라면 클론이 응답한다.
8. 발신자는 동일 대화창에서 사람 메시지와 AI 메시지를 함께 본다.
9. 발신자 또는 수신자는 자신의 화면에서 AI 개입 묶음을 숨길 수 있다.
10. 사용자는 필요 시 텍스트 메시지를 음성으로 듣거나 음성 메시지형 대화를 이용한다.

## 13. 핵심 기능 상세

### 피드

- 사용자는 게시물을 업로드할 수 있다.
- 공개 계정의 게시물은 누구나 조회할 수 있다.
- 비공개 계정의 게시물은 이미 팔로우 중인 사용자만 조회할 수 있다.

### 프로필

- 사용자는 자신의 프로필을 편집할 수 있다.
- 프로필에서 게시물, 클론/보이스 설정, 논쟁 기능에 접근한다.

### 팔로우

- 공개 계정은 팔로우 가능하다.
- 비공개 계정은 신규 팔로우 요청이 불가능하다.
- 맞팔로우는 관계 의미상 기존 친구 개념을 이어받을 수 있다.

### DM

- DM은 1:1 대화다.
- 기본 메시지 타입은 텍스트다.
- DM 대상은 기본적으로 사람이다.
- 자동응답이 켜진 경우 클론이 대신 응답한다.

### 클론 설정

- 클론은 사용자 대리 AI다.
- 초기 생성 방식은 설문 기반이다.
- 사용자당 1개만 유지한다.

### 보이스 설정

- 보이스는 사용자 클론의 대표 음성이다.
- 사용자당 1개만 유지한다.
- 텍스트 응답에 대해 음성 재생을 제공할 수 있다.

### 논쟁 기능

- 논쟁은 메인 탭이 아니라 프로필에서 접근하는 부가 기능이다.
- 논쟁은 제품의 핵심 사용 흐름이 아니다.

### Settings

- `Settings`는 개인 설정 성격의 항목을 모은다.
- 공개성 설정은 `Settings`에서 관리한다.
- 자동응답 설정은 `Settings`에서 관리한다.
- AI 숨김은 DM 안에서 묶음 단위로 조작하고, `Settings`는 해당 정책과 복원 위치를 안내한다.

## 14. AI/클론 동작 정책

- 클론은 사용자 자신을 대리한다.
- DM 자동응답에서만 클론 응답이 개입한다.
- 자동응답 설정:
  - 항상: 사용자가 접속 여부와 무관하게 클론이 DM에 자동응답한다.
  - 내가 접속중이지 않을 때: 마지막 API 요청 후 3분이 지나면 클론이 자동응답한다.
  - 사용 안함: 클론 자동응답을 하지 않는다.
- AI 응답은 일반 메시지와 동일한 대화 흐름에 쌓이지만, 시각적으로 명확히 구분된다.
- AI 응답은 숨김 기능의 대상이 될 수 있다.
- AI는 AI가 생성한 메시지에 응답하지 않는다.
- MVP에서 점진적 학습은 지원하지 않는다.

## 15. 계정 공개성 및 관계 정책

- 계정 공개성은 `공개`, `비공개` 두 가지다.
- 공개 계정은 검색 가능하다.
- 공개 계정의 공개 게시물은 누구나 조회 가능하다.
- 공개 계정은 누구나 DM 가능하다.
- 비공개 계정은 검색 노출이나 상세 노출 정책을 별도로 조정할 수 있으나, MVP 기준으로 핵심 제약은 게시물 및 DM 접근 제한이다.
- 비공개 계정은 이미 팔로우 중인 사용자만 게시물 조회 및 DM 가능하다.
- 비공개 계정은 신규 팔로우 요청이 불가능하다.
- 맞팔로우는 상호 관계의 확장 개념으로만 해석하며, 별도 친구 도메인을 두지 않는다.

## 16. DM 정책

- DM은 텍스트 기반 1:1 대화가 기본이다.
- 공개 계정은 누구나 DM을 시작할 수 있다.
- 비공개 계정은 이미 팔로우 중인 사용자만 DM을 시작할 수 있다.
- DM 한 대화창 안에 사람 메시지와 AI 메시지가 함께 저장된다.
- AI 응답에는 `AI` 시각 표시가 필요하다.
- AI 응답을 유발한 사람 메시지와 AI 응답은 하나의 숨김 묶음으로 취급한다.
- AI 숨김은 개인 UI 설정이므로 다른 사용자에게 영향을 주지 않는다.

예시:

- A가 B에게 DM을 보낸다.
- B가 부재중이며 자동응답이 켜져 있다.
- B의 클론이 응답한다.
- A가 자신의 UI에서 해당 AI 응답을 숨기면, A가 보낸 유발 메시지와 B 클론의 응답이 함께 보이지 않게 된다.

## 17. 음성 기능 정책

- DM의 기본 상호작용은 텍스트다.
- 텍스트 DM 메시지는 아이콘을 눌러 음성으로 들어볼 수 있다.
- 별도의 음성 대화 모드를 제공한다.
- 음성 대화 모드는 실시간 통화가 아니라 음성 메시지형 대화다.
- 음성 대화 모드에서 AI가 응답할 경우 음성으로 응답하는 것이 기본이다.
- 텍스트 DM에서 생성된 AI 응답도 필요 시 음성 재생을 지원할 수 있다.

## 18. 데이터 및 상태 정책

- 부재중 판정에 필요한 마지막 활동 시각을 저장해야 한다.
- 마지막 활동 시각은 모든 API 요청을 기준으로 갱신한다.
- 메시지 저장 시 사람 작성 메시지와 AI 작성 메시지를 구분할 수 있어야 한다.
- AI 메시지는 표시용 메타데이터를 가져야 한다.
- AI 숨김은 개인 UI 상태이므로 사용자별 숨김 상태 저장 구조가 필요하다.
- 사용자별 클론 1개, 보이스 1개 제약을 데이터 모델에서 보장해야 한다.
- 점진적 학습용 데이터 누적은 장기적으로 고려 가능하지만 MVP 스키마에 필수는 아니다.

## 21. 결정 사항 요약

- 서비스 본체는 SNS다.
- DM 기본 대상은 사람이다.
- 클론은 사용자 대리 AI다.
- 사용자당 클론 1개, 보이스 1개다.
- 자동응답 설정은 `항상 / 내가 접속중이지 않을 때 / 사용 안함`이다.
- 부재중 기준은 마지막 API 요청 후 3분이다.
- 활동은 모든 API 요청으로 정의한다.
- 공개 계정은 누구나 DM 가능하다.
- 비공개 계정은 신규 팔로우 요청이 불가능하고, 이미 팔로우 중인 사용자만 게시물 조회 및 DM 가능하다.
- AI 응답은 대화창에 함께 저장되지만 `AI` 마크로 표시한다.
- AI 숨김은 개인 UI 설정이며, 유발 메시지와 AI 응답을 묶어서 숨긴다.
- AI는 AI 메시지에 자동응답하지 않는다.
- DM은 텍스트 기반이며, 텍스트를 음성으로 재생할 수 있다.
- 음성 대화는 음성 메시지형 대화다.
- 점진 학습은 MVP에서 제외한다.
- 논쟁 기능은 프로필에서 접근하는 부가 기능이다.
