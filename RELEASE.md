# Release Notes

## 2026-03-26

이번 업데이트에서는 회원 인증, 사용자 소유권 분리, 프론트 인증 흐름, 테스트 보강, 개발용 기본 계정 부트스트랩까지 한 번에 정리했다.

## 주요 변경

### 1. 회원 인증 도입

- JWT access token 기반 회원가입, 로그인, 로그아웃, 내 정보 조회 추가
- 회원 엔티티에 soft delete 적용
- 탈퇴 후 기존 토큰으로 보호 API 접근 차단

### 2. 사용자 소유권 분리

- 클론, 등록 음성, 채팅 대화, 논쟁 세션, 생성 오디오 자산에 `user` 소유권 연결
- 모든 보호 API가 현재 로그인 사용자 기준으로만 조회/사용 가능하도록 변경
- 다른 사용자의 클론, 음성, 대화, 논쟁에 접근할 수 없도록 백엔드 검증 추가

### 3. 백엔드 API 보호

- `/api/system-prompt`
- `/api/clones`
- `/api/voices`
- `/api/chat/**`
- `/api/debates/**`

위 경로들에 JWT 인증과 소유권 검증을 적용했다.

### 4. 프론트 인증 흐름 추가

- 로그인/회원가입 화면 추가
- 앱 시작 시 `/api/auth/me`로 세션 복구
- `Authorization: Bearer ...` 자동 부착
- `401` 응답 시 자동 로그아웃
- 계정 바에 현재 사용자, 로그아웃, 회원 탈퇴 액션 추가

### 5. 사용자 기준 프론트 상태 정리

- 클론/음성/대화/논쟁이 현재 로그인 사용자 기준으로만 보이도록 정리
- 사용자 전환 시 기존 라이브 세션, 오디오, 모달, 입력 상태 초기화
- 페이지를 실제로 떠날 때만(`pagehide`) 스트림과 재생 정리

### 6. 토론 흐름 개선

- 기존 `stop` API 제거
- 논쟁 종료는 클라이언트가 다음 턴 요청을 멈추는 방식으로 단순화
- 토론 컨텍스트에 주제가 반드시 포함되도록 수정
- 고정 찬성/반대 역할 없이, 각 클론의 성격 프롬프트에 따라 자유롭게 토론하도록 변경

### 7. OpenAI 호출 공통화

- `PromptService`, `ChatService`, `DebateService`에 흩어져 있던 OpenAI 호출 패턴을 공통 클라이언트 계층으로 정리
- HTTP 요청, 응답 파싱, 공통 예외 처리를 한 곳으로 모음

### 8. 개발용 기본 계정 부트스트랩

- 설정으로 활성화하면 서버 시작 시 지정한 기본 계정이 없을 때 자동 생성
- 기본 계정과 함께 기본 클론 2개, 기본 음성 2개 자동 준비
- 기본 클론은 테스트용 시스템 프롬프트까지 함께 저장
- 음성 샘플 경로는 절대경로/상대경로 모두 지원
- 상대경로는 현재 작업 디렉터리 우선, 없으면 `backend` 기준으로 재해석

## 운영/개발 시 유의점

- JWT secret은 32바이트 이상 권장
- soft delete 된 계정은 동일 아이디로 자동 재생성하지 않음
- 기본 음성 부트스트랩은 실제 DashScope 음성 등록을 수행하므로 API 키와 샘플 파일이 필요
- 오래전에 다른 DashScope 모델로 등록한 음성은 현재 TTS 모델과 호환되지 않을 수 있음
- 스트리밍 연결 종료 시 서버 로그에 클라이언트 연결 중단 `IOException`이 보일 수 있으며, 브라우저 abort 상황에서는 정상 범주일 수 있음
